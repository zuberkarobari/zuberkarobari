## 🏗️ System Design Portfolio

> Architecture-first engineering — every project below is designed with HLD + LLD rigour.

---

### 🏦 J.P. Morgan UPI Payment System v2

```
Architecture:  Lean Modular Monolith (9 Modules)
Stack:         Java 21 · Spring Boot 3.2 · PostgreSQL · Redis · PhonePe UPI 2.0
Pattern:       Domain-Driven Design + Event-Driven Internal Bus
```

**Key Design Decisions:**

- 🔒 **ACID Compliance** — `@Transactional(SERIALIZABLE)` + `SELECT FOR UPDATE` row-level locks preventing race conditions on ledger debits/credits
- 📨 **Internal Event Bus** — 5 named `Spring ApplicationEvents` decoupling 9 modules without Kafka overhead for atomic internal transactions
- 🛡️ **Idempotency Layer** — Redis (24h TTL) + SHA256 webhook checksum validation → zero double-charges, zero payload spoofing
- 📅 **Automated Reconciliation** — `@Scheduled` cron jobs fetching PSP daily reports, executing EOD break-detection against internal settlement records

---

### 💳 VISA Real-Time Settlement Engine

```
Architecture:  Event-Driven High-Throughput Pipeline
Stack:         Java · Spring Boot · Apache Kafka · Spring Batch
Pattern:       CQRS-adjacent + Exactly-Once Processing
```

**Key Design Decisions:**

- 📊 **High-Volume Reconciliation** — 50K+ daily settlements · ~99.99% transaction accuracy
- ⚡ **Kafka Partitioning** — Consumer group partitioning + idempotent producer + exactly-once semantics → zero duplicate customer notifications
- 🚀 **Parallel Batch Processing** — Re-engineered legacy sequential batch using Java parallel streams + connection pool tuning → 67% execution time reduction (6 min → 2 min)
- 🔄 **Data Format Unification** — OpenCSV (CSV→POJO) + JAXB (XML→POJO) pipeline reducing new data feed onboarding from 3 days → 4 hours

---

### 👷 GetWork — Hyperlocal Gig Economy Platform

```
Architecture:  Two-Sided Marketplace + Distributed Wallet Engine
Stack:         Spring Boot 3.2 · Firebase Cloud Functions · React Native 0.81 · PostgreSQL · Firestore
Domain:        Blue-collar gig economy · Construction & Trades · Hyderabad launch
```

**Key Design Decisions:**

- 🔄 **State Machine** — 6-stage bidirectional task lifecycle: `Open → Applied → Assigned → In-Progress → Completion_Requested → Completed`
- 💰 **Atomic Ledger** — Immutable append-only ledger transactions with 10% platform commission auto-deductions on job completion
- 🗺️ **Geospatial Engine** — Haversine formula proximity matching + Geohash (Precision 6) caching for Google Maps API route/distance results
- 💵 **Dynamic Pricing** — Real-time budget calculators: city-specific base pay + travel allowances + dynamic worker count adjustments

### 🪨 Karobari Stone Management System

```
Architecture:  Industrial Logistics Suite + Digital Ledger
Stack:         Spring Boot · React · Firebase · Custom Calculation Engine
Domain:        Stone factory trip/payment/logistics tracking
```

**Key Design Decisions:**

- 📒 **Digital Ledger** — Digitised complex physical ledger workflows with industrial-familiar UI
- 💳 **Prepaid Wallet Model** — Worker credit wallet system replacing direct employer billing; auto-debit on job completion
- 🧮 **Logistics Math Engine** — Automated stone production output calculations (`nag × 3` = footage), truck expense tracking, trip-by-trip reconciliation

### 🤖 TrackR v2 — AI Finance App

Stack:         Expo · Zustand · MMKV · Claude Vision API · Firebase Cloud Functions
